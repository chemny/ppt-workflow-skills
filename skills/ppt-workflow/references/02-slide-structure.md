# 02 Slide Structure

Use this workflow to design the story, chapters, page sequence, and slide-by-slide structure.

## When to use

Use when the user needs an outline, story flow, deck structure, page count, section arrangement, or a clearer logic chain.

## Inputs

- Goal brief.
- Existing notes, documents, data, or draft slides.
- Target page count and presentation length.
- Audience and scenario.

## Process

1. Choose a narrative pattern that fits the goal:
   - Problem -> insight -> solution -> action.
   - Context -> analysis -> recommendation -> next steps.
   - Current state -> gap -> plan -> impact.
   - Question -> evidence -> answer.
   - Teaching path: concept -> example -> practice -> summary.
2. Create chapters before creating individual slides.
3. Give each slide one job and one core message.
4. Assign rough timing per slide.
5. Mark which slides need data support.
6. Flag weak logic, missing bridges, or overloaded sections.

## Output format

```markdown
## Slide Structure
### Narrative
[One-sentence story line]

### Chapters
| Chapter | Purpose | Slides | Time |
|---|---|---:|---:|

### Slide Plan
| # | Slide title | Purpose | Core message | Needs data? | Time |
|---:|---|---|---|---|---:|

### Logic Risks
- [Risk or missing link]
```

## Handoff

- Send slides marked "Needs data" to `data-research`.
- Send confirmed slide plan to `slide-writing`.
