# 03 Data Research

Use this workflow to identify what evidence is needed, find data sources, and support key claims.

## When to use

Use when the deck contains claims, comparisons, market sizing, performance results, financial assumptions, industry facts, customer insights, or any conclusion that needs proof.

## Inputs

- Slide structure.
- Claims or key messages.
- Industry, region, timeframe, and source constraints.
- User-provided files or links.
- Whether web research is allowed.

## Process

1. Extract claims that require support.
2. Define required evidence for each claim.
3. Set source standards:
   - Primary source when possible.
   - Official reports, filings, datasets, company docs, credible research, or user-provided internal data.
4. Search or inspect sources when allowed.
5. Cross-check important numbers.
6. Mark unsupported, weak, outdated, or risky claims.
7. Convert findings into PPT-ready conclusions.

## Output format

```markdown
## Data Research Plan
| Slide | Claim | Needed data | Recommended source | Finding | Confidence | Citation | Risk |
|---:|---|---|---|---|---|---|---|

## PPT-ready Conclusions
- [Slide #]: [Conclusion with source]

## Claims to Weaken or Reword
- [Claim]: [Reason]
```

## Handoff

Send findings to `slide-writing`. If evidence is missing, ask the user whether to continue with placeholders or revise the claim.
