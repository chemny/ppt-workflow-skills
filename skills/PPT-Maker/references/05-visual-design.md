# 05 Visual Design

Use this workflow to define the deck's visual direction, brand rules, page layout, chart style, and image style.

## When to use

Use when the user wants a better-looking PPT, a company style, a template, brand consistency, chart styling, image direction, or redesign advice.

## Inputs

- Slide writing plan or existing PPT.
- Brand guidelines, template PPT, reference deck, website, or natural language style request.
- Audience and scenario.
- Constraints and forbidden styles.

## Process

1. Identify the deck type: executive report, sales proposal, training, technical talk, academic defense, pitch, product launch, etc.
2. Define a reusable visual design spec.
3. Choose page density appropriate to the audience.
4. Define layout rules: title, section pages, data pages, comparison pages, process pages, closing pages.
5. Define chart rules before styling charts.
6. Define image rules: subject, mood, composition, color, copyright risk.
7. Keep visual choices consistent across pages.
8. Avoid visual changes that weaken data accuracy.

## Output format

```markdown
## Visual Design Spec
- Deck type:
- Visual direction:
- Colors:
- Fonts:
- Layout grid:
- Title style:
- Section page style:
- Data/chart style:
- Image style:
- Icon/illustration style:
- Do not use:

## Page Design Notes
| Slide | Layout type | Visual treatment | Asset needs | Risk |
|---:|---|---|---|---|
```

## Handoff

Send the visual design spec and page notes to `ppt-building`.
